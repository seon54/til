# 운영체제 발전 역사



## 1950년대 

- ENIAC : 첫 번째 컴퓨터

  - 운영체제 X
  - 1개의 응용 프로그램 실행
  - 응용 프로그램이 시스템 자원 관리

  

## 1960년대 초기

	> 프로그램 종류와 사용자가 많아짐

 - 배치 처리 시스템(Batch Processing system) 출현
    - 여러 응용 프로그램을 등록하면 순차적으로 실행
    - 배치 처리 시스템을 기반으로 운영체제 출현



## 1960년대 후기

- 시분할 시스템(Time Sharing System)
  - 다중 사용자 지원
  - 컴퓨터 응답 시간 최소화
- 멀티 태스킹(Multi Tasking)
  - 단일 CPU에서 여러 응용 프로그램의 병렬 실행을 가능하게 함

> 응용 프로그램이 CPU를 사용하는 시간을 쪼개서 여러 응용 프로그램을 동시에 실행하는 기법. **구현되지는 않음**

> 멀티 프로그래밍 : 최대한 CPU를 많이 활용하도록 하는 시스템(시간대비 CPU 활용도 높이기)



### 배치 처리 시스템의 단점

1. 컴퓨터 응답 시간이 오래 걸릴 수 있다.
2. 실행 시간이 오래 걸릴 수 있다.(CPU가 필요 없어도 응용 프로그램이 CPU를 점유할 수 있으므로)



### 시분할 시스템과 배치 처리 시스템

- 시분할 시스템 / 멀티 태스킹
  - 핵심 기술 : 시간을 잘게 쪼개서 여러 응용 프로그램 실행
  - 컴퓨터 응답 시간 줄임(시분할 세스템 : 다중 사용자 지원)
  - 전체 응용 프로그램의 실행 시간도 줄임(멀티 프로그래밍)
  - 사용자가 느끼기에 여러 응용 프로그램이 동시에 실행되는 것처럼 보임(멀티 태스킹)
